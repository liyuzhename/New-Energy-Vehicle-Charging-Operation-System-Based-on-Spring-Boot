# 前端问题说明

本文档记录后端修复后，前端需要对应调整的内容。

---

## 问题1：用户端钱包流水按类型搜索无反应

**后端修改：** `GET /api/user/wallet/records` 新增可选查询参数 `type`。

**前端需要修改：**

调用流水记录接口时，将用户选择的类型作为 `type` 参数传入：

```js
// 示例
GET /api/user/wallet/records?type=RECHARGE&page=1&size=10
GET /api/user/wallet/records?type=CONSUME&page=1&size=10
GET /api/user/wallet/records?page=1&size=10  // 不传 type 则返回全部
```

`type` 可选值：
- `RECHARGE`：充值
- `CONSUME`：消费
- `REFUND`：退款

## 问题2：用户端每隔一段时间弹出"当前没有进行中的充电订单"

**后端修改：** `GET /api/order/charging` 在无进行中订单时不再抛异常，改为返回 `{"code":200,"data":null}`。

**前端需要修改：**

轮询该接口时，判断 `data` 是否为 `null`，为 `null` 时静默处理，不弹出错误提示：

```js
// 修改前（错误做法）
if (res.code !== 200) {
  showToast(res.message) // 会弹出"当前没有进行中的充电订单"
}

// 修改后（正确做法）
if (res.code === 200 && res.data !== null) {
  // 更新充电状态
} else {
  // 无充电订单，静默处理，不弹窗
}
```

## 问题3：用户端订单列表字段为空（充电站、充电时长、充电量、金额）

**后端修改：** `GET /api/order/my` 返回的每条 `OrderVO` 新增以下字段：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `stationName` | String | 充电站名称 |
| `pileNo` | String | 充电桩编号 |
| `power` | BigDecimal | 充电桩功率（kW） |
| `plateNo` | String | 车牌号 |
| `chargeDuration` | Long | 充电时长（秒），已完成订单为实际时长，充电中为当前已充时长 |

**前端需要修改：**

订单列表直接使用上述新字段渲染，无需前端自行关联查询：

```js
// 充电站名称
item.stationName

// 充电时长（秒转分钟/小时显示）
const minutes = Math.floor(item.chargeDuration / 60)

// 充电量
item.chargeKwh

// 金额
item.totalFee

// 车牌号
item.plateNo
```

## 问题4：充电页面充电桩/充电站/实时功率为空，已充电量和预估费用不更新

**后端修改：** `GET /api/order/charging` 返回的 `OrderVO` 新增以下字段（实时计算）：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `stationName` | String | 充电站名称 |
| `pileNo` | String | 充电桩编号 |
| `power` | BigDecimal | 充电桩功率（kW） |
| `plateNo` | String | 车牌号 |
| `chargingSeconds` | Long | 已充电时长（秒） |
| `chargeKwh` | BigDecimal | 已充电量（kWh），实时按功率×时长计算 |
| `estimatedFee` | BigDecimal | 预估费用（元），按计费规则实时计算 |
| `chargeDuration` | Long | 同 `chargingSeconds` |

**前端需要修改：**

充电页面轮询 `GET /api/order/charging`，直接使用返回的字段渲染，无需前端自行计算：

```js
// 充电站
page.stationName = res.data.stationName

// 充电桩
page.pileNo = res.data.pileNo

// 实时功率
page.power = res.data.power + ' kW'

// 已充电量
page.chargeKwh = res.data.chargeKwh + ' kWh'

// 预估费用
page.estimatedFee = res.data.estimatedFee + ' 元'

// 充电时长
page.duration = formatSeconds(res.data.chargingSeconds)
```

```json
}
```

**前端需要修改：**

- `orderId` 为 Long 类型，不能为字符串

```js

  orderId: order.id,      // Long
  rating: form.rating,    // Integer 1-5，必填
  content: form.content   // String，可选
}
```

## 问题12：运营商端统计报表数据为空

**后端接口：** `GET /api/report/order-trend`、`GET /api/report/pile-usage`，后端数据正常。

**后端修改：** 两个接口均新增可选 `stationId` 参数，支持按充电站过滤。

**前端需要修改：**

1. 充电站下拉框选中后，将充电站 `id` 作为 `stationId` 传入：

```
GET /api/report/order-trend?stationId=15&startDate=YYYY-MM-DD&endDate=YYYY-MM-DD
GET /api/report/pile-usage?stationId=15&startDate=YYYY-MM-DD&endDate=YYYY-MM-DD
```

2. 按维度（日/周/月）搜索：前端根据选择的粒度计算 `startDate/endDate` 传给后端：

```js
// 按日（今天）
startDate = dayjs().format('YYYY-MM-DD')
endDate   = dayjs().format('YYYY-MM-DD')

// 按周（本周一到周日）
startDate = dayjs().startOf('week').add(1, 'day').format('YYYY-MM-DD')  // 周一
endDate   = dayjs().endOf('week').add(1, 'day').format('YYYY-MM-DD')    // 周日

// 按月（本月）
startDate = dayjs().startOf('month').format('YYYY-MM-DD')
endDate   = dayjs().endOf('month').format('YYYY-MM-DD')
```

`order-trend` 返回结构：
```json
[{ "period": "2026-03-04", "orderCount": 2 }]
```

`pile-usage` 返回结构：
```json
[{ "pileId": 3, "orderCount": 5, "occupiedMinutes": 360 }]
```
