# 基于 Spring Boot 的新能源汽车充电桩运营管理系统——功能设计文档

---

## 一、系统概述

本系统是一套面向新能源汽车充电服务场景的**纯软件业务管理平台**，采用**前后端分离**架构：

- **后端**：基于 Spring Boot + MyBatis-Plus 构建 RESTful API 服务，负责业务逻辑处理与数据持久化
- **前端**：基于 Vue 3 + Element Plus 构建单页应用（SPA），通过 Axios 调用后端接口，实现数据展示与交互

系统通过完整的业务闭环设计，覆盖充电桩日常运营全链路，包括：用户注册与车辆绑定 → 查桩与预约 → 发起充电（模拟）→ 实时计费 → 订单结算与钱包支付 → 评价反馈 → 运营统计报表，无需与真实硬件直接交互，所有充电状态与数据均由后台业务逻辑驱动。

**系统 MVC 分层说明：**

| 层次 | 对应实现 | 职责 |
|------|----------|------|
| **Model（模型层）** | Spring Boot：Entity + Service + Mapper | 数据封装、业务逻辑处理、数据库交互 |
| **View（视图层）** | Vue 3 + Element Plus | 页面渲染、用户交互、数据展示 |
| **Controller（控制层）** | Spring MVC Controller（返回 JSON） | 接收前端请求、调用 Service、统一返回 JSON 响应 |

---

## 二、核心目标

- **业务闭环**：从用户找桩到支付完成，形成完整的服务链路，每个节点均有对应功能模块支撑。
- **多角色协作**：普通用户、站点运营商、超级管理员三类角色职责分离，基于 JWT + Spring Security 实现接口权限隔离。
- **数据可追溯**：所有充电行为、订单状态、费用计算均有完整记录，支持历史查询与对账。
- **运营可视化**：运营商和管理员可通过报表直观掌握充电桩利用率、收入趋势、故障情况。
- **无硬件依赖**：充电桩状态（空闲/占用/故障）、充电数据（电量/时长）均通过业务逻辑管理，运营商可手动维护，系统提供完整模拟充电流程。

## 五、核心模块详细功能

> 所有接口统一返回 JSON 格式，后端使用 `@RestController`，前端通过 Axios 调用。  
> 接口基础路径：`/api`，Token 在请求头中携带：`Authorization: Bearer <token>`

### 5.1 用户中心模块

**面向角色：** 所有角色

**Vue 页面：** 登录页 `/login`、注册页 `/register`、个人中心 `/user/profile`

| 功能 | 请求方式 + 路径 | 说明 |
|------|-----------------|------|
| 用户注册 | `POST /api/auth/register` | 提交用户名、**手机号（必填）**、**邮箱（必填）**、密码（6-20位），默认角色 USER |
| 用户登录 | `POST /api/auth/login` | 校验账号密码，返回 JWT Token，前端存入 localStorage |
| 忘记密码（发送验证码） | `POST /api/auth/forgot-password` | 提交注册邮箱，后端通过 **QQ SMTP** 发送 6 位随机验证码，存入 **Redis**（有效期 5 分钟） |
| 重置密码 | `POST /api/auth/reset-password` | 提交邮箱、验证码、新密码（6-20位），校验 Redis 中验证码后更新密码并删除 Redis key |
| 获取当前用户信息 | `GET /api/user/info` | 前端携带 Token 请求，返回当前登录用户信息 |
| 修改个人信息 | `PUT /api/user/profile` | 修改昵称、头像、手机号 |
| 修改密码 | `PUT /api/user/password` | 提交旧密码与新密码，BCrypt 校验后更新 |
| 查看绑定车辆 | `GET /api/user/vehicles` | 返回该用户所有已绑定车辆列表 |
| 绑定车辆 | `POST /api/user/vehicles` | 提交 `plateNo`（车牌号）、品牌、型号、电池容量 |
| 解绑车辆 | `DELETE /api/user/vehicles/{id}` | 删除指定车辆绑定记录 |
| 查看钱包 | `GET /api/user/wallet` | 返回余额、累计充值、累计消费 |
| 钱包充值 | `POST /api/user/wallet/recharge` | 提交充值金额，模拟支付直接到账，生成充值流水 |
| 查看流水记录 | `GET /api/user/wallet/records` | 分页返回充值、消费、退款所有流水记录 |

**注册字段校验规则（变更）：**
- `username`：3-20 位，必填
- `phone`：1[3-9]开头11位手机号，必填，唯一
- `email`：邮箱格式，必填，唯一
- `password`：6-20 位，必填

**忘记密码流程：**
1. 前端提交邮箱 → `POST /api/auth/forgot-password`
2. 后端检查邮箱是否注册 → 生成 6 位验证码 → 存入 Redis（key: `reset:code:{email}`，5分钟过期）→ 通过 QQ SMTP 发送邮件
3. 前端填写验证码+新密码 → `POST /api/auth/reset-password`
4. 后端校验 Redis 验证码 → 更新密码 → 删除 Redis key

**前端交互要点：**
- 登录成功后 JWT Token 存入 `localStorage`（记住密码）或 `sessionStorage`（不记住），Axios 请求拦截器自动在 Header 中携带
- Token 过期后前端路由守卫拦截并跳转至登录页
- 使用 Element Plus 的 `ElForm` 组件完成表单校验

### 5.4 充电服务（订单）模块

**面向角色：** 普通用户（发起）、运营商（查看名下订单）、管理员（全量查看）

**Vue 页面：** 发起充电 `/charging/start`、充电中 `/charging/progress/:orderId`、订单列表 `/user/orders`

| 功能 | 请求方式 + 路径 | 说明 |
|------|-----------------|------|
| 发起充电 | `POST /api/order/start` | 提交桩编号 + 车辆 ID，校验桩为 IDLE 后创建订单，桩状态变为 OCCUPIED |
| 查询充电中订单 | `GET /api/order/charging` | 返回当前用户进行中的订单信息（时长、预估费用） |
| 停止充电 | `PUT /api/order/{orderId}/stop` | 记录结束时间，计算电量与分段费用，生成账单，桩状态恢复 IDLE |
| 订单详情 | `GET /api/order/{orderId}` | 返回完整账单：时长、电量、峰平谷分段费用明细、总金额 |
| 我的订单列表 | `GET /api/order/my` | 分页返回历史订单，支持按状态/时间筛选 |
| 运营商订单列表 | `GET /api/operator/order/list` | 运营商查看名下充电站所有订单，支持分页与筛选 |
| 全平台订单列表 | `GET /api/admin/order/list` | 管理员查看全平台所有订单，支持按 `status`、`orderNo`（模糊）、`startDate`/`endDate` 日期范围筛选 |
| 申请退款 | `POST /api/order/{orderId}/refund` | 已支付订单申请退款，订单状态变为 REFUNDING |
| 退款审核通过 | `PUT /api/admin/order/{id}/refund/approve` | 管理员审核通过，退款至钱包，状态变为 REFUNDED |
| 退款审核拒绝 | `PUT /api/admin/order/{id}/refund/reject` | 管理员拒绝退款，订单状态恢复 PAID |

**充电费用计算逻辑：**

1. 充电时长 = `end_time - start_time`（单位：小时）
2. 充电电量 = 充电桩额定功率（kW）× 充电时长（h）
3. 按充电时段匹配峰/平/谷计费规则
4. 若跨越多个时段，按比例拆分各时段电量分别计费
5. 总费用 = Σ（各时段电量 × 电费单价）+ Σ（各时段电量 × 服务费单价）

**前端交互要点：**
- 充电进行中页面使用 `setInterval` 每秒刷新时长显示，定时轮询接口获取预估费用
- 使用 Element Plus `ElSteps` 组件展示订单状态流转步骤
- 账单详情页使用表格展示分段计费明细

## 六、业务闭环流程

```
用户在 Vue 前端登录（JWT Token 返回并存入 localStorage）
        ↓
  绑定名下车辆（Element Plus 表单提交 → POST /api/user/vehicles）
        ↓
  浏览充电站列表（GET /api/station/list → Vue 卡片列表展示）
  → 查看充电站详情与计费规则
        ↓
  预约充电桩（选日期+时段 → POST /api/reservation，可选步骤）
        ↓
  输入桩编号 → 发起充电（POST /api/order/start）
  后端校验桩状态为 IDLE，创建订单，桩状态变为 OCCUPIED
        ↓
  充电进行中（前端轮询 GET /api/order/charging，实时展示时长与预估费用）
        ↓
  点击「停止充电」（PUT /api/order/{id}/stop）
  后端计算电量 = 功率 × 时长，按峰平谷拆分生成账单
        ↓
  确认支付（POST /api/order/{id}/pay）
  后端扣减钱包余额，订单状态变为 PAID，桩状态恢复 IDLE
        ↓
  跳转评价页，提交星级评价（POST /api/evaluation）
        ↓
  运营商查看收益报表（GET /api/report/income）
  管理员查看全平台仪表盘（GET /api/admin/dashboard）
```
