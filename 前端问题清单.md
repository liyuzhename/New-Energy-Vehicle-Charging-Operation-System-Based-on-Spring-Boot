## 问题1：记住密码功能错误 / 未勾选时频繁弹出"登录已过期"

**现象：**
不勾选"记住密码"的情况下，点击个人信息、我的车辆、我的报修等页面，会弹出"登录已过期"提示。

**期望行为：**
- 勾选"记住密码"：下次打开浏览器无需重新输入账号密码，token 持久化保存（存入 `localStorage`）。
- 不勾选"记住密码"：本次会话有效，关闭浏览器后需重新登录，token 存入 `sessionStorage`。

**问题根因：**
前端登录逻辑中，无论是否勾选"记住密码"，token 均存入 `sessionStorage`（或未做区分），导致页面刷新或跳转后 token 丢失，触发 401 拦截器弹出过期提示。

**修复建议：**
在登录成功回调中根据"记住密码"复选框状态分别存储 token：
```js
if (rememberMe) {
  localStorage.setItem('token', token)
} else {
  sessionStorage.setItem('token', token)
}
```
读取 token 时同时检查两处：
```js
const token = localStorage.getItem('token') || sessionStorage.getItem('token')
```
退出登录时两处均清除。

## 问题2：后台用户管理启用/禁用报"缺少必要参数：status"

**现象：**
在后台用户管理页面点击启用或禁用按钮时，弹出提示"缺少必要参数：status"。且列表默认显示全部用户为禁用状态，与实际数据不符。

**后端接口：**
```
PUT /api/admin/user/{id}/status?status=0   // 禁用
PUT /api/admin/user/{id}/status?status=1   // 启用
```
后端 `parseStatus()` 支持 `0/1`、`disabled/enabled`、`禁用/启用` 多种格式，接口本身正常。

**问题根因：**
1. 前端调用启用/禁用接口时，未将 `status` 作为 query 参数传递（可能放在了 body 中，或参数名拼写错误）。
2. 用户列表默认显示禁用状态，说明前端渲染 status 字段时逻辑有误（如将 `1` 显示为禁用，`0` 显示为启用，逻辑反了）。

**修复建议：**
1. 调用接口时确保 status 作为 query 参数传递：
```js
axios.put(`/api/admin/user/${id}/status`, null, { params: { status: newStatus } })
```
2. 列表渲染时：`status === 1` 为启用（绿色），`status === 0` 为禁用（红色/灰色）。

## 问题3：后台故障管理时间搜索按钮无反应（前端对接部分）

**现象：**
点击故障管理页面的时间搜索按钮后，列表数据没有变化。

**后端接口（已修复）：**
```
GET /api/admin/fault/list?status=&startDate=2024-01-01&endDate=2024-12-31&page=1&size=10
GET /api/operator/fault/list?status=&startDate=2024-01-01&endDate=2024-12-31&page=1&size=10
```
后端已新增 `startDate` 和 `endDate` 参数（ISO 日期格式 `yyyy-MM-dd`），支持时间范围过滤。

**问题根因：**
前端搜索按钮点击事件未将日期选择器的值拼接到请求参数中，或日期格式与后端不匹配。

**修复建议：**
搜索时将日期选择器的值格式化为 `yyyy-MM-dd` 后作为参数传入：
```js
const params = {
  status: searchStatus,
  startDate: startDate ? dayjs(startDate).format('YYYY-MM-DD') : undefined,
  endDate: endDate ? dayjs(endDate).format('YYYY-MM-DD') : undefined,
  page: currentPage,
  size: pageSize
}
axios.get('/api/admin/fault/list', { params })
```

## 问题4：注册页面缺少邮箱输入框 / 个人信息页面无邮箱字段

**现象：**
前端注册表单中没有邮箱输入框，个人信息页面也没有显示邮箱字段。

**后端接口：**
注册接口 `POST /api/auth/register` 的请求体中 `email` 字段标注了 `@NotBlank` 和 `@Email`，是**必填项**。若前端不传 email，注册请求会被后端校验拦截返回 400 错误。

**修复建议：**
1. 注册表单中添加邮箱输入框，并做前端格式校验（`/^[^\s@]+@[^\s@]+\.[^\s@]+$/`）：
```html
<el-form-item label="邮箱" prop="email">
  <el-input v-model="form.email" placeholder="请输入邮箱" />
</el-form-item>
```
2. 个人信息页面添加邮箱字段的展示与编辑。

## 问题5：运营商收益统计页面报错"缺少必要参数：startDate"

**现象：**
进入运营商端收益统计页面后，控制台报错：
```
Uncaught (in promise) Error: 缺少必要参数：startDate
    at request.interceptors.response.use.status (request.js:28:29)
    at async Axios.request (Axios.js:41:14)
    at async loadData (OperatorIncome.vue:114:17)
```

**后端接口：**
```
GET /api/report/income?startDate=2024-01-01&endDate=2024-01-31
```
后端 `startDate` 和 `endDate` 均为 `required = false`，有默认值（近30天），后端不会报此错误。

**问题根因：**
`request.js` 的响应拦截器中对某些接口做了额外的参数校验，当响应数据为空或特定格式时误判为"缺少必要参数"。或者 `OperatorIncome.vue` 的 `loadData` 方法在组件挂载时未传 `startDate`/`endDate` 参数，触发了前端自定义的参数校验逻辑。

**修复建议：**
1. 检查 `request.js` 第28行附近的拦截器逻辑，确认是否有对 `startDate` 的前端校验，如有则移除或修正。
2. 在 `OperatorIncome.vue` 的 `loadData` 方法中，确保初始调用时传入默认日期范围：
```js
const endDate = dayjs().format('YYYY-MM-DD')
const startDate = dayjs().subtract(29, 'day').format('YYYY-MM-DD')
await loadData(startDate, endDate)
```
