# 新能源汽车充电桩运营管理系统 · 后端安装使用说明

> 本文档描述**后端服务**的环境准备、安装部署与基本使用步骤。  
> 本系统采用**前后端分离**架构：后端提供 REST API，前端通过 HTTP 调用后端接口。  
> 请与前端项目的《安装使用说明》合并后，作为完整系统的使用手册。

---

## 一、项目概述

| 项目 | 说明 |
|------|------|
| 项目名称 | 新能源汽车充电桩运营管理系统（后端） |
| 技术栈 | Spring Boot 3.2.5 + MyBatis-Plus + MySQL 8.0 + Redis + Spring Security + JWT |
| 开发语言 | Java 21 |
| 构建工具 | Maven 3.8+ |
| 默认服务端口 | `8090` |
| 接口前缀 | `/api` |
| 主启动类 | `com.charging.ChargingApplication` |

### 1.1 前后端协作关系

```
┌─────────────┐      HTTP/JSON       ┌─────────────────────────────┐
│  前端 Web   │  ─────────────────►  │  后端 Spring Boot (8090)    │
│  (Vue 等)   │  ◄─────────────────  │  REST API + JWT 认证        │
└─────────────┘   Authorization      └──────────┬──────────────────┘
                                                │
                                    ┌───────────┴───────────┐
                                    │  MySQL (charging_db)  │
                                    │  Redis (验证码缓存等)  │
                                    └───────────────────────┘
```

- 后端默认允许以下来源的跨域请求：`http://localhost:5173`、`http://localhost:5174`（常见 Vue 开发端口）。
- 前端开发时，需将 API 基地址配置为 `http://localhost:8090`。
- 除登录、注册等公开接口外，其余请求需在 Header 中携带 JWT Token。

| 软件 | 最低版本建议 | 用途 |

## 三、软件安装

以下以 **Windows** 为例，其他操作系统请使用对应包管理器或官方安装包。

### 3.1 安装 JDK 21

1. 从 [Oracle JDK](https://www.oracle.com/java/technologies/downloads/) 或 [Adoptium Temurin](https://adoptium.net/) 下载 JDK 21 并安装。
2. 配置环境变量 `JAVA_HOME` 指向 JDK 安装目录，并将 `%JAVA_HOME%\bin` 加入 `Path`。
3. 打开命令行验证：

```cmd
java -version
```

预期输出包含 `21` 版本信息。

### 3.2 安装 Maven

1. 从 [Apache Maven 官网](https://maven.apache.org/download.cgi) 下载并解压。
2. 配置环境变量 `MAVEN_HOME`，并将 `%MAVEN_HOME%\bin` 加入 `Path`。
3. 验证：

```cmd
mvn -version
```

### 3.3 安装并启动 MySQL

1. 安装 MySQL 8.0，安装过程中设置 root 密码（后续配置文件中会用到）。
2. 确保 MySQL 服务已启动，默认端口 `3306`。
3. 验证连接：

```cmd
mysql -u root -p
```

### 3.4 安装并启动 Redis

1. Windows 可使用 [Redis for Windows](https://github.com/microsoftarchive/redis/releases) 或通过 WSL / Docker 运行 Redis。
2. 启动后默认监听 `6379` 端口。
3. 验证（若已安装 redis-cli）：

```cmd
redis-cli ping
```

预期返回 `PONG`。

## 五、数据库初始化

### 5.1 执行顺序

**首次部署（推荐）：**

| 步骤 | 脚本 | 说明 |
|------|------|------|
| 1 | `init.sql` | 创建数据库 `charging_db`、全部数据表、初始管理员账号 |
| 2 | `fix_test_data.sql` | 导入完整测试数据（运营商、用户、充电站、订单等） |

**可选：**

- 若仅需基础测试数据，可执行 `test_data.sql` 代替 `fix_test_data.sql`。
- 若数据库为旧版本且缺少站点字段，可额外执行 `add_station_fields.sql`。

### 5.2 执行命令（Windows CMD）

在项目根目录下执行（按提示输入 MySQL root 密码）：

```cmd
mysql -u root -p --default-character-set=utf8mb4 < src\main\resources\sql\init.sql
mysql -u root -p --default-character-set=utf8mb4 < src\main\resources\sql\fix_test_data.sql
```

**PowerShell 等价命令：**

```powershell
Get-Content src\main\resources\sql\init.sql | mysql -u root -p --default-character-set=utf8mb4
Get-Content src\main\resources\sql\fix_test_data.sql | mysql -u root -p --default-character-set=utf8mb4
```

### 5.3 初始化结果

- 数据库名：`charging_db`
- 字符集：`utf8mb4`
- 默认管理员：`admin` / 密码 `admin123`
- 更多测试账号详见项目内 [测试账号说明.md](./测试账号说明.md)

### 5.4 重新初始化（清空后重建）

如需完全重置数据，请先删除数据库再重新执行上述脚本：

```sql
DROP DATABASE IF EXISTS charging_db;
```

然后重新执行 `init.sql` 与 `fix_test_data.sql`。

## 六、配置文件说明

编辑 `src/main/resources/application.yml`，根据本机环境修改以下配置。

### 6.1 服务端口

```yaml
server:
  port: 8090
```

若端口被占用，可改为其他端口（需同步修改前端 API 地址）。

### 6.2 数据库连接

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/charging_db?useUnicode=true&characterEncoding=utf8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    username: root
    password: 你的MySQL密码
```

### 6.3 Redis 连接

```yaml
spring:
  data:
    redis:
      host: localhost
      port: 6379
      database: 0
```

### 6.4 邮件服务（忘记密码功能，可选）

```yaml
spring:
  mail:
    host: smtp.qq.com
    port: 587
    username: 你的邮箱@qq.com
    password: 邮箱SMTP授权码
```

> 若不配置邮件，注册、登录、充电等核心功能仍可正常使用；仅「忘记密码 / 邮箱验证码」相关功能不可用。

### 6.5 JWT 配置

```yaml
jwt:
  secret: 请替换为足够长的随机字符串
  expiration: 86400000   # Token 有效期 24 小时（毫秒）
```

> **安全提示：** 生产环境务必修改 `jwt.secret` 和数据库密码，不要将含真实密钥的配置文件提交到公开仓库。

## 七、编译与启动

### 7.1 命令行启动（推荐）

在项目根目录执行：

```cmd
# 1. 下载依赖并编译（首次或依赖变更后执行）
mvn clean package -DskipTests

# 2. 启动应用
mvn spring-boot:run
```

或直接运行打包后的 JAR：

```cmd
java -jar target\charging-1.0.0.jar
```

### 7.2 IDE 启动（IntelliJ IDEA）

1. 使用 IDEA 打开项目（识别为 Maven 项目）。
2. 等待 Maven 依赖下载完成。
3. 找到 `src/main/java/com/charging/ChargingApplication.java`。
4. 右键 → **Run 'ChargingApplication'**。

### 7.3 启动成功标志

控制台出现类似日志即表示启动成功：

```
Started ChargingApplication in x.xxx seconds
Tomcat started on port 8090 (http)
```

此时后端基地址为：`http://localhost:8090`

## 九、与前端联调

### 9.1 启动顺序

1. 启动 **MySQL** 与 **Redis**
2. 启动 **后端**（本说明，端口 8090）
3. 启动 **前端**（参考前端安装说明，常见端口 5173）

### 9.2 前端需配置的内容

前端项目通常需配置 API 代理或环境变量，例如：

```
VITE_API_BASE_URL=http://localhost:8090
```

具体变量名以前端项目为准。

### 9.3 跨域说明

后端已在 `SecurityConfig` 中配置 CORS，默认放行：

- `http://localhost:5173`
- `http://localhost:5174`

若前端使用其他端口，需在 `src/main/java/com/charging/config/SecurityConfig.java` 的 `corsConfigurationSource()` 方法中追加对应地址后重新编译启动。

## 十、角色与权限说明

系统包含三种角色，接口按路径前缀区分权限：

| 角色 | 标识 | 主要接口前缀 | 功能范围 |
|------|------|-------------|---------|
| 普通用户 | `USER` | `/api/user/**`、`/api/order/**` 等 | 充电、订单、钱包、预约、评价 |
| 运营商 | `OPERATOR` | `/api/operator/**`、`/api/report/**` | 充电站/桩管理、计费、故障处理 |
| 管理员 | `ADMIN` | `/api/admin/**` | 用户管理、公告、全局统计、操作日志 |

未登录访问受保护接口返回 `401`；角色不匹配返回 `403`。

## 十四、常见问题排查

| 现象 | 可能原因 | 处理建议 |
|------|---------|---------|
| 启动报数据库连接失败 | MySQL 未启动、密码错误、库未创建 | 检查 MySQL 服务；确认已执行 `init.sql`；核对 `application.yml` 中的账号密码 |
| 启动报 Redis 连接失败 | Redis 未启动 | 启动 Redis 服务，确认 host/port 正确 |
| 端口 8090 被占用 | 其他程序占用端口 | 修改 `server.port` 或结束占用进程 |
| 前端请求 401 | Token 未携带或已过期 | 重新登录获取 Token；检查 Header 格式 `Bearer <token>` |
| 前端请求 403 | 当前角色无权访问 | 使用对应角色账号登录 |
| 前端跨域报错 | 前端端口不在 CORS 白名单 | 在 `SecurityConfig` 中添加前端地址 |
| 登录密码错误 | 数据库未导入测试数据或密码 hash 不一致 | 重新执行 `fix_test_data.sql` |
| `java -version` 不是 21 | JDK 版本不匹配 | 安装 JDK 21 并正确配置 `JAVA_HOME` |
| Maven 依赖下载慢 | 网络或镜像源问题 | 配置国内 Maven 镜像（如阿里云） |
